Prueba de pantalla completa simple

Mismo enfoque CSS base: viewport + body max-width + padding

Esta pagina no carga React, no tiene navbar, no tiene bottom nav, no usa contenedores internos y no usa estilos globales de la app.

El objetivo es probar si esta estructura plana es la que hace que una pagina se perciba ocupando toda la pantalla en mobile.

Codigo usado

Viewport: width=device-width, initial-scale=1.0

Body: padding: 20px; max-width: 980px; margin: auto;

Fondo: background: #111 aplicado directamente al body.

Contenido de prueba

Si en Safari mobile esta pagina se ve como ContactosSW, entonces la clave no es una API de fullscreen sino la estructura HTML plana: el body es el layout principal.

Si esta pagina se sigue viendo con el mismo marco o cuadrado que Mansion Deseo, entonces el efecto no viene de estas tres reglas y hay que buscar otra diferencia del navegador, del modo de apertura, del scroll o de la configuracion del sitio.
